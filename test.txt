1. Big Picture – What the Project Is
You are building one C++ program that:

Reads a file called airports.csv (it has real flight data: 384 flights between 140 airports).
Turns the data into a directed graph (airports = nodes, direct flights = edges). Each edge stores two numbers: Distance and Cost.
Lets the user do 8 specific things with the graph (listed below).
Uses graph algorithms you implement yourself (Dijkstra, Prim’s, Kruskal’s, etc.).

You cannot use most of the C++ Standard Template Library (STL). The only STL thing you are allowed to use for data structures is std::vector. You must code your own Graph, queues, priority queues, etc.
2. What You Must Submit (Deliverables)

Source code: All your .cpp and .h files (as many as you need). No ZIP files, no binaries, no extra folders.
Report (one file): Must contain
Description of every class you created.
Time complexity (Big-O) for every algorithm you implemented.
Sample runs of your program (including weird edge cases like “no path exists”).

Presentation: 3–4 minutes long. You will run your program live and show it working.
Code must compile and run perfectly on: https://www.onlinegdb.com/online_c++_compiler

Team size: 2–3 people. Submit together.
3. The 8 Things Your Program Must Actually Do
Here is every required feature, written in super-simple language:

Read the CSV and build the graph

Load “airports.csv”. Create a directed graph where each flight is an edge with two weights (Distance + Cost).
Shortest path between two specific airports
User gives origin airport and destination airport (e.g., IAD to MIA).
Output the path, total length (distance), and total cost.
If no path: show a clear “None” message.
(Sample: “Shortest route from IAD to MIA: IAD -> ORD -> MIA. The length is 765. The cost is 234.”)
All shortest paths from one airport to every airport in a whole state
User gives origin airport + state (e.g., ATL to all airports in FL).
Show every path + length + cost in a nice table.
If none exist: clear message.
Shortest path with a given number of stops
User gives origin, destination, and number of stops (e.g., 3 stops).
Find the shortest path that has exactly that many intermediate airports.
Output path, length, and cost.
If impossible: clear message.
Count total direct connections for every airport
For each airport, count:
Inbound flights (arriving)
Outbound flights (leaving)
Total = inbound + outbound
Print a sorted list: airports with the most connections first.
(Sample shows a simple table with ATL 12, ORD 11, etc.)

Build an undirected version of the graph (G_u)
Take your original directed graph and turn it into an undirected one using these exact rules:
If there is only one direction between two airports → keep that edge and use its Cost (ignore Distance).
If there are flights both ways → keep only the cheaper (minimum Cost) one as an undirected edge (ignore Distance).

Prim’s algorithm – Minimum Spanning Tree on the undirected graph
Run Prim’s on G_u (use Cost as the weight to minimize).
Output every edge in the MST + total cost.
If the graph is disconnected → say an MST cannot be formed.
Kruskal’s algorithm – Minimum Spanning Tree (or Forest) on the undirected graph
Run Kruskal’s on G_u (use Cost as the weight).
Output every edge + total cost.
If disconnected → build a “minimum spanning forest” (one MST for each separate group of airports).

4. Quick Implementation Tips (High-Level Plan)

Start by creating a Graph class (using vectors only).
Store airports as strings or IDs.
Each edge needs to remember: destination airport, Distance, and Cost.
Implement the algorithms one by one (you’ll need your own versions of priority queue for Dijkstra/Prim, Union-Find for Kruskal, etc.).
Write clear comments in the code explaining what each function does.
Test with the sample outputs shown in the PDF (they are fake examples, but match the exact format they want).
